# Glossary

Terms specific to this project's documentation that are not immediately obvious. The dydo
system's own vocabulary — sprint, slice, lane, role, skill, agent, and the rest — is locked
in the [dydo glossary](./reference/dydo-glossary.md); this file covers what's left.

---

## How to Use

Scan for the term you hit in the docs. If it isn't here, check the
[dydo glossary](./reference/dydo-glossary.md) first, then the folder's hub file.

---

## Terms

### Hub File

An `_index.md` file in each documentation folder. Serves as the table of contents for that
folder. Auto-generated by `dydo fix`.

### JITI

Just-In-Time Information. The documentation philosophy: agents don't read everything
upfront — they navigate to what they need for the current task.

### Off-Limits

File patterns in `dydo/files-off-limits.md` that block ALL agent access — secrets,
credentials, system files. A whitelist section can carve exceptions.

### Template Addition

A markdown file in `dydo/_system/template-additions/` injected into templates via
`{{include:name}}` tags. Allows extending compiled skills without editing templates
directly.
