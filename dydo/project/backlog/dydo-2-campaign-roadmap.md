## Target model (after this Campaign)

- **Two tiers.** *Tier 1* = named agents you talk to (orchestrator / co-thinker / single), launched in terminals, onboard via `dydo agent claim`. *Tier 2* = anonymous native sub-agents (code-writer / reviewer / test-writerâ€¦), identified by `agent_type` + `agent_id`, spawned by workflows or the Tier-1 agent. No names, no claim. See [024 Â§1](../decisions/024-dydo-2-native-pivot.md).
- **Guard = universal policy layer.** Off-limits (hard) + nudges (soft), keyed on `agent_type`/`agent_id`, fires for sub-agents *and* workflow workers (verified). Per-role path RBAC is gone.
- **Access scope = native.** Permission profiles (`tools` / `disallowedTools` / `permissionMode`) per agent-type; a few shipped defaults (`read-only`, `writer`, `trusted`). Not dydo-rolled.
- **Roles split, don't vanish.** Tier-1 collapses to 3 modes (orchestrator / co-thinker / single); Tier-2 worker functions become native agent-types + skills generated by `dydo sync`.
- **Runtime slimmed, not retired.** KEEP: top-level dispatch, top-levelâ†”top-level messaging (light), watchdog (now hosts Notion sync), docs tooling, audit (re-keyed on `agent_id`), tasks/issues. GONE: worker-tier dispatch/inbox/queue/worktree-dispatch.

## Flagship workflow: `run-sprint`

The core saved workflow (`.claude/workflows/run-sprint.js`). Shape from the human's normal loop:

```
slice sprint into disjoint units
  â””â”€ per slice (pipeline, parallel across slices):
       code-writer (worktree) â†’ reviewer
         loop:  reviewer fails â†’ new code-writer with feedback â†’ reviewer
         exit conditions (whichever first):
           - reviewer PASSES                          â†’ slice done
           - 5 failed review rounds (hard cap)        â†’ escalate to human
           - any agent pulls the "raise hand" signal  â†’ escalate to human early
merge passed slices sequentially (conflicts escalate to human)
return structured sprint result to the orchestrator â†’ human re-engages
```

### Escalation rules (both required)

1. **Hard cap â€” 5 failed review rounds.** Grounded in experience: up to 4 consecutive natural fails observed in the wild (each reviewer pass surfacing a new real issue). 5 gives headroom without burning tokens on a stuck slice.
2. **Soft circuit-breaker â€” "raise hand."** Any code-writer or reviewer can proactively break the loop and escalate when it senses something is off and human eyes are needed (ambiguous spec, architectural smell, repeated thrash on the same root cause). Mechanism TBD in Sprint 4 â€” candidates: a sentinel in structured output the workflow checks each round, or a dedicated nudge/marker.
