# OrphanDocsRule flags transient task files until dydo fix runs - adds noise on every check after agent role-set

`Rules/OrphanDocsRule` scopes `project/tasks/` in via `MainDocFolders` and BFS-checks reachability through the auto-generated `tasks/_index.md`, so every newly-created task file is flagged as an orphan until `dydo fix` runs. `FrontmatterRule` already gives task files special treatment via a separate schema; `OrphanDocsRule` is the only rule that still treats them as first-class permanent docs, producing 13 task-orphan warnings on this project alone.

## Description

Rules/OrphanDocsRule.cs:13 - MainDocFolders = ["guides", "project", "reference", "understand"]. Task files live under project/tasks/, so the rule scopes them in. Rules/OrphanDocsRule.cs:38-43 - BFS-reachability from project/_index.md. A task only counts as reachable when tasks/_index.md lists it. tasks/_index.md is auto-regenerated by dydo fix (Commands/FixHubHandler.RegenerateHubs -> Services/HubGenerator.GenerateHub) - the auto-gen banner makes this explicit.\n\nResult: every newly-created task file is an orphan until dydo fix runs. Live reproduction: 13 task-orphan warnings on dydo project, including dydo-check-drift-ruling.md created when I (Dexter) set my role at 17:55 UTC. LC reproduces on 2 task files plus 1 orphan resolved-issue file (different cause - see issue for finding #8 / LC's queue).\n\nFrontmatterRule:36-50 already gives task files special treatment (different frontmatter schema: name/status/created/assigned). OrphanDocsRule is the only rule that still treats task files as first-class permanent docs.\n\nFix path (preferred): add 'if (normalized.StartsWith("project/tasks/", StringComparison.OrdinalIgnoreCase) && !doc.FileName.StartsWith("_")) return [];' at the top of OrphanDocsRule.Validate. Mirrors FrontmatterRule:36-50 exactly. Side-effect: task files no longer need to appear in tasks/_index.md - dydo fix could either keep listing them (status quo) or stop listing them (cleaner; the index file becomes meaningful only for the description).\n\nAlternative (not recommended): trigger hub regen on task-file create/touch. Higher coupling, more moving parts.\n\nOpen question for planner (same nudge as issues for findings #1 and #6): should the four-element MainDocFolders array be configurable / data-driven?\n\nConfirmed by inquisition dydo-check-drift.md finding #4 (judge: Dexter).
