# v1.3.0 Release Plan

Stabilize v1.2 features and evolve the framework based on real-world usage. Everything ships together, developed in parallel.

---

## Slices

### A — Guard Hardening
Fix sub-agent hook evasion and pre-claim read restrictions. The guard is the backbone — if it's porous, everything built on top is theater.

- [ ] Investigate how Agent tool sub-processes bypass PreToolUse hooks
- [ ] Ensure unclaimed agents can only read entry-point files
- [ ] Verify guard enforcement across all agent spawn paths

### B — Messaging / Wait / Dispatch Fixes
Make the features shipped in v1.2 actually work.

- [ ] dispatch --wait background enforcement
- [ ] inbox clear state sync
- [ ] reply-pending guardrail
- [ ] auto-close behavior
- [ ] Clear stuck waiting states (Brian, Dexter)

### C — Workflow & Role Revamp
Align the documented workflow with how the system is actually used. See [decision 005](decisions/005-fresh-agent-over-wait-for-feedback.md), [decision 006](decisions/006-drop-interviewer-role.md), and plan at `agents/Iris/plan-v1.3-workflow-role-revamp.md`.

#### C1 — Workflow template + role docs
- [ ] Simplify workflow template to 3 steps (claim → orient → work)
- [ ] Keep `--inbox` as only flag, drop all others
- [ ] Drop interviewer role (absorbed by co-thinker)
- [ ] Rename tester → test-writer, update mode file
- [ ] Update co-thinker mode file (primary entry point, scoping techniques)
- [ ] Update all mode file templates for new workflow

#### C2 — New roles: orchestrator + inquisitor
- [ ] Orchestrator mode file (swarm coordination, dispatch + monitor)
- [ ] Inquisitor mode file (adversarial QA, hypothesis-driven)
- [ ] Guard: orchestrator is graduation-only (not dispatchable)
- [ ] Guard: `dispatch --wait` restricted to orchestrator + inquisitor

#### C3 — `dispatch --worktree`

#### C5 — Inquisition coverage

### F — Release Documentation

- [ ] This release plan

### G — Terminal Grouping + Agent Tree
Fix tab chaos in multi-agent workflows. See [decision 004](decisions/004-terminal-grouping-over-dashboard.md).

- [ ] `wt -w 0` in TerminalLauncher.cs (same-window tab spawning)

### H — TerminalLauncher Platform Fixes
`--new-window` dispatch fails with `error 2147942402 (0x80070002)` — "The system cannot find the file specified." The launch command uses `powershell` which may not resolve on Windows 11 (needs `pwsh` or full path). Terminals still open despite the error.

- [ ] Fix `powershell` resolution in launch command (pwsh vs powershell, full path)
- [ ] Audit TerminalLauncher.cs for platform-specific assumptions
- [ ] Test `--new-window` and `--tab` on Windows 11
- [ ] Verify macOS launch commands
