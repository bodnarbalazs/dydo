# v1.3.0 Release Notes

Multi-agent orchestration, messaging, worktree isolation, and a simplified workflow.

---

## Highlights

### Multi-Agent Orchestration

Agents can now coordinate other agents. The **orchestrator** role dispatches work, monitors progress via `dydo wait`, and manages feedback loops. Dispatch chains allow natural handoffs: orchestrator dispatches code-writer, code-writer dispatches reviewer, reviewer reports back.

### Messaging System

Direct agent-to-agent communication via `dydo msg` and `dydo wait`. Messages carry task context, enable async coordination, and integrate with the reply-pending guardrail to ensure dispatched work gets reported back upstream.

### Worktree Dispatch

`dispatch --worktree` creates isolated git worktrees so parallel agents work on separate branches without conflicts. Child dispatches inherit the parent's worktree. Cleanup happens automatically on agent release.

### Simplified Workflow

The agent workflow is now three steps: **claim -> orient -> work**. All workflow flags (`--feature`, `--task`, `--quick`, `--think`, `--review`, `--docs`, `--test`) are dropped. Agents infer their role from the prompt. The only flag is `--inbox`, used when an agent is dispatched by another agent.

---

## New Roles

| Role | Purpose |
|------|---------|
| **orchestrator** | Coordinates multi-agent workflows and task dispatch |
| **inquisitor** | Conducts adversarial QA and knowledge audits |
| **judge** | Evaluates inquisition reports and arbitrates disputes |

The `interviewer` role was dropped (absorbed by `co-thinker`). The `tester` role was renamed to `test-writer`.

## New Features

### Dispatch Queue
Named queues (`--queue <name>`) serialize terminal launches. A watchdog monitors queue state and advances deferred items when the active agent releases. Useful for merge queues and other sequential workflows.

### Issue Tracker
`dydo issue create/list/resolve` ‚Äî lightweight issue management at `dydo/project/issues/`. Integrates with inquisitor pipeline (confirmed findings become issues) and judge arbitration.

### Inquisition Coverage
`dydo inquisition coverage` ‚Äî file-level heatmaps tracking which areas have been audited. Rolling per-area reports at `dydo/project/inquisitions/`.

### Custom Nudges
Project-specific coaching hints in `dydo.json`. Regex-matched against tool inputs with `block` or `warn` severity. Extends the guard system with conventions that aren't covered by base guardrails.

### Conditional Must-Reads
Dynamic role-specific documentation requirements. Merge guides for worktree code-writers, task files for reviewers ‚Äî injected at runtime based on context.

### Custom Roles
User-defined roles via `.role.json` files in `dydo/_system/roles/`. `dydo roles create <name>` scaffolds a new role. Custom roles participate in all guard checks like base roles.

### Template Additions
`{{include:name}}` tags in templates resolve to markdown files in `dydo/_system/template-additions/`. Inject project-specific steps without editing templates directly. Preserved across `dydo template update`.

### Agent Tree
`dydo agent tree` shows the dispatch hierarchy of active agents as an ASCII tree.

## Guard & Infrastructure

- **Guard hardening** ‚Äî Sub-agent hook evasion fixed, pre-claim read restrictions enforced
- **Base64/HEREDOC detection** ‚Äî Guard detects bypass attempts via encoding and heredocs
- **EnterPlanMode block** ‚Äî Prevents agents from entering plan mode (bypasses guard)
- **Guard lift/restore** ‚Äî `dydo guard lift <agent>` temporarily suspends RBAC for human-directed work
- **Dispatch --wait enforcement** ‚Äî `--wait` restricted to oversight roles (orchestrator, inquisitor, judge)
- **Double-dispatch protection** ‚Äî Blocks dispatching to a task that already has an active agent
- **Auto-close improvements** ‚Äî Watchdog-based, handles non-shell processes
- **Process leak fixes** ‚Äî Parent PID liveness checks prevent zombie agents
- **Zombie worktree cleanup** ‚Äî `git worktree prune` on release

## Platform & Terminal

- **Terminal grouping** ‚Äî Dispatched agents open in tabs within the same window (Windows Terminal, macOS)
- **Platform-specific launchers** ‚Äî Fixed `powershell` resolution on Windows 11, verified macOS
- **Window inheritance** ‚Äî Child dispatches inherit `DYDO_WINDOW` for tab grouping

## Workflow Changes

- Workflow simplified to 3 steps: claim -> orient -> work
- All workflow flags dropped except `--inbox`
- `interviewer` role removed (absorbed by `co-thinker`, see [decision 006](decisions/006-drop-interviewer-role.md))
- `tester` renamed to `test-writer`
- `co-thinker` is now the default role for direct human interaction
- Orchestrator requires graduation (prior `co-thinker` or `planner` on the same task)
- Judge has panel limit (max 3 per task)
