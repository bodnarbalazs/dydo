# v1.4.0 Release Notes

A wait-paradigm shift, auto-resume on crash, and a small fix tail on top of v1.3.9.

---

## Highlights

### Unified General Wait

Every agent — not just orchestrators — now keeps an always-active general wait, registered immediately after the role is set. The per-task `dydo wait --task <name>` pattern is dropped from the standard workflow (the CLI form remains for special cases). This closes the orchestrator general-wait deadlock that recurred whenever an inbox message arrived between rearms.

The `dispatch --wait` privilege stays oversight-only, but its meaning shifts: it is now a *release-block* on the dispatched agent (cannot release until they have messaged the dispatcher back on the task subject), not a blocking call on the caller. The caller continues working; their general wait surfaces the reply when it lands.

See [Decision 021](decisions/021-unified-general-wait.md).

### Auto-Resume on Crash

When a claude session dies mid-task while `state.md` still says `working`, the watchdog now relaunches the same agent with `claude --resume <sessionId>` and a short continuation prompt — capped at three retries per crashed session. Default-on for any agent with a captured `.session` file; no flag.

Identity, role, task, and workspace are preserved by construction: the same session ID short-circuits `HandleExistingSession`, so the resumed claude rejoins as the same dydo agent rather than triggering the stale-working reclaim path.

See [Decision 022](decisions/022-auto-resume-crashed-agents.md).

## Bug Fixes

- **Auto-close kill regression restored** — `ReleaseAgent` no longer clears `auto-close` prematurely. The per-agent claim lock added in v1.3.9 already closed the redispatch race that motivated the original change, so the post-release `free + auto-close: true` window the watchdog needs is back. Released agents' terminals close again. (#0134)
- **Auto-resume launches in the project root** — the resume launcher now passes the project root as `workingDirectory` instead of the user home. (#0138)
- **General-wait registration is idempotent** — wait snapshot aligned with the inbox dir; re-registering an active general wait no longer trips the guard. (#0141)

---

## Workflow Changes

- General wait is mandatory for **all** roles — register `dydo wait` (no `--task`) in the background right after `dydo agent role`. Workflows that skip this will block on the next tool call.
- `dydo dispatch` no longer auto-registers a `dydo wait --task <name>` for the dispatcher. Existing scripts using the per-task form still work but are redundant against the universal general wait.
- `dispatch --wait` enforces a release-block on the dispatched agent, not a blocking call on the dispatcher. Every mode template now reflects the new pattern.
